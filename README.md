# Terrix

![images](img.jpg)

Terrix 是一个面向 DEM 结果对比与可视化分析的桌面工具。适合用来比较多个模型输出的地形栅格结果，把同名 `tif/tiff` 样本自动对齐后进行并排预览、局部放大、坡度/坡向派生分析、误差图查看，以及高分辨率图片导出。

## 适合做什么

- 对比多个 DEM 超分/重建/生成模型的输出结果
- 以同一参考模型为基准检查各模型在局部区域的表现
- 查看高程、坡度、坡向三种地形表达结果
- 分析模型与参考结果之间的误差分布
- 从 DEM 中提取河网并叠加到预览图上
- 导出论文、汇报或实验记录需要的高分辨率对比图

## 主程序功能

### 1. 多模型目录管理

- 支持添加多个模型结果目录
- 支持设置参考
- 支持模型排序、重命名、修改路径、删除
- 支持从 Finder 直接拖入模型文件夹
- 自动记住上一次保存的模型列表和浏览路径

### 2. 自动匹配样本

- 自动扫描每个目录中的 `tif/tiff`
- 只保留所有模型目录中共同存在的同名样本
- 通过样本下拉框和“上一个 / 下一个”快速切换文件

### 3. 多视图地形分析

- 支持三种显示模式：
  - 高程
  - 坡度
  - 坡向
- 坡度与坡向基于 Horn 梯度计算
- 可手动设置：
  - 像元大小 X / Y
  - `Z factor`
  - 坡向平坦区域是否置 0

### 4. 并排预览与局部放大

- 所有模型结果按统一布局并排显示
- 每个模型同时展示整图和局部放大图
- 可拖动放大框，在不同模型间同步查看同一局部区域
- 可调节放大框大小、模型标题字号、色带字号和放大框颜色

### 5. 误差图分析

- 支持切换“显示误差图”
- 支持三种误差指标：
  - `ME`
  - `MAE`
  - `RMSE`
- 误差图会以参考为基准计算
- 对坡向误差做了环状角度差处理，避免 0/360 度边界带来的失真

### 6. 色带与可视化控制

- 内置多种色带，包括 `Terrain`、`Viridis`、`Turbo`、`Jet`、`Gray`、`Magma`、`Plasma`、`Seismic`、`HSV` 等
- 支持色带反向
- 自动绘制颜色条与数值刻度

### 7. 河网叠加

- 可直接从 DEM 中提取 D8 河网并叠加在预览图上
- 支持是否先进行填洼（Priority-Flood）
- 可设置流量阈值、河网线宽、河网点大小和河网颜色
- 可同时显示整图和局部放大区域中的河网结果

### 8. 高分辨率导出

- 支持导出当前对比图
- 支持格式：
  - `png`
  - `jpg`
  - `pdf`
  - `tif`
- 支持自定义 `72-1200` DPI

## 独立子工具：River Network Eval

仓库内的 `river-network-eval` 是一个独立的河网评估 GUI，可用于批量计算模型河网与参考河网之间的 `mIoU(%)`。

它提供：

- 参考 DEM 目录与模型目录管理
- 按相对路径匹配同名 `tif/tiff`
- 基于 D8 + 汇流阈值提取河网
- 可选填洼
- 运行进度与日志显示
- CSV 结果导出
- 最后一行输出各模型平均值 `AVERAGE`

这个工具适合把“可视化观察”进一步变成“批量量化评估”。

## 输入数据要求

- 输入数据为单波段 `tif/tiff` DEM 栅格
- 各模型目录下应包含同名样本，便于自动匹配
- 做并排对比时，建议各模型输出具有一致的尺寸与对应关系
- 坡度、坡向、河网提取依赖像元大小和高程比例参数，因此这些参数需要按数据实际情况设置

## 运行方式

### 主程序

```bash
cd terrix-mac-1.0.0
./gradlew run
```

### 独立河网评估工具

```bash
cd terrix-mac-1.0.0
./gradlew -p river-network-eval run
```

## 打包

主程序使用 Java 17 + Gradle 构建，仓库中已经提供 macOS 打包任务：

```bash
cd terrix-mac-1.0.0
./gradlew packageMacApp
./gradlew packageMacDmg
```

## 技术特点

- Java 17
- Swing 桌面 GUI
- 基于本地 TIFF 栅格直接读取与渲染
- 无需依赖大型 GIS 软件即可完成多模型 DEM 结果对比
- 面向实验分析流程，强调“快速浏览 + 局部检查 + 可视化导出 + 定量评估”

## 一句话总结

Terrix 不是单纯的 DEM 查看器，它更像一个围绕“模型结果对比”设计的轻量桌面工作台：既能看整体，也能看局部；既能看高程，也能看坡度、坡向、误差和河网；既能导出展示图，也能补上河网 mIoU 的量化评估。
